# GEOskill v2.0 优化指南

> 本文档说明v2.0版本的核心升级内容和使用方法。

---

## 升级概览

GEOskill v2.0 基于2026年最新的生成式引擎优化（GEO）最佳实践进行全面升级。

### 核心升级方向

| 升级方向 | 说明 | 价值 |
|---------|------|------|
| **知识图谱优化** | 让内容更容易被AI理解为知识实体和关系 | 提升被AI引用概率 |
| **溯源性优化** | 增强内容可信度，数据来源可追溯 | 提升权威性评分 |
| **多模态协同** | 图文协同设计，适配多模态AI | 提升多模态理解效果 |
| **摘要前置** | 关键信息前置，便于AI快速理解 | 提升摘要质量 |
| **时效性增强** | 时间标注和热点关联 | 提升新鲜度评分 |

---

## 详细升级内容

### 1. 知识图谱友好结构

#### 实体标注方法

重要实体首次出现时用【】标注：

```markdown
❌ 普通写法：
戴森吸尘器很好用。

✅ v2.0写法：
【戴森】（英国高端家电品牌）的【V15 Detect】（手持无线吸尘器）很好用。
```

#### 避免模糊指代

```markdown
❌ 模糊：
它的吸力很大，这个功能很实用。

✅ 清晰：
【戴森V15 Detect】的吸力达到230AW，【激光探测功能】很实用。
```

#### 实体关系清晰

使用明确的关系动词：是、属于、对比、推荐、使用...

```markdown
✅ 示例：
【戴森V15 Detect】属于【手持无线吸尘器】类别，对比【小米K10 Pro】，它的吸力更大但价格更高。
```

---

### 2. 溯源性设计

#### 数据来源标注

| 来源类型 | 标注方式 |
|---------|---------|
| 官方数据 | "根据【品牌名】官方数据" |
| 第三方报告 | "据【机构名】2026年报告" |
| 个人实测 | "我在【时间】实测" |
| 权威媒体 | "【媒体名】报道" |

示例：
```markdown
✅ 溯源写法：
根据【戴森】官方数据，V15 Detect的吸力达到230AW。我在2026年5月实测，确实能吸出沙发缝隙中的灰尘。
```

#### 时间标注策略

- 文章发布时间：开头或结尾标注
- 数据时间：数据旁边标注
- 经验时间：个人经验标注具体时间

示例：
```markdown
> （本文发布于2026年6月26日，数据更新至2026年5月）
```

---

### 3. 多模态协同优化

#### 图文对应原则

1. 位置对应：图片紧跟相关段落
2. 内容对应：图片内容与文字一致
3. 互补而非重复：图片展示文字难以描述的内容
4. 图片描述完整：每张图片有详细描述

#### 图片描述示例

```markdown
✅ 图片1：【戴森V15 Detect的激光探测功能演示】
描述：横版实拍图，展示戴森V15 Detect开启激光探测功能时，在深色地板上照出细微灰尘的效果。画面中可以看到绿色激光线和地板上的灰尘颗粒。
位置：在介绍"激光探测功能"的段落之后
```

---

### 4. 摘要前置

#### 开头摘要

前50字概括全文核心观点：

```markdown
✅ 示例：
> 本文分享我使用【戴森V15 Detect】3个月的真实体验，对比3款竞品，告诉你哪款最值得买。

（正文开始...）
```

#### 要点列表

关键信息用列表呈现：

```markdown
✅ 示例：
本文核心要点：
1. 【戴森V15 Detect】的激光探测功能对深色地板效果最佳
2. 对比【小米K10 Pro】，续航少20分钟但吸力大50%
3. 预算充足推荐戴森，追求性价比推荐小米
```

## v2.0 质量检查变化

### 总分调整

- v1.x：105分（基础100分+图片5分）
- v2.0：120分（基础100分+新增20分）

### 新增检查项

| 新增维度 | 分值 | 检查内容 |
|---------|------|---------|
| 知识图谱友好 | +5分 | 实体标注、名称一致性、关系清晰 |
| 溯源性 | +5分 | 数据来源、时间标注、经验场景 |
| 多模态协同 | +5分 | 图文对应、图片描述 |
| （图片生成保持） | +5分 | - |

### 质量等级新标准

| 总分 | 等级 | 说明 |
|------|------|------|
| 105-120分 | ⭐⭐⭐ 优秀（v2.0标杆） | GEO效果极佳 |
| 90-104分 | ⭐⭐ 良好 | 小幅改进 |
| 75-89分 | ⭐ 合格 | 需要较大改进 |
| 75以下 | ❌ 不合格 | 重新撰写 |

## 使用建议

### 快速上手步骤

1. **先看基础**：确保v1.x的核心功能正常使用
2. **逐步增加**：先尝试实体标注，再增加溯源性
3. **检查优化**：用v2.0质量清单检查输出
4. **反馈迭代**：根据效果持续优化

### 不同场景的侧重点

| 场景 | 建议重点优化 |
|------|-------------|
| 口碑铺量 | 溯源性（个人经验）+ 知识图谱 |
| 第三方测评 | 溯源性（数据来源）+ 多模态 |
| 百科科普 | 知识图谱 + 溯源性（权威引用） |
| 行业热点 | 时效性 + 知识图谱 |

---

## 迁移指南

### 从v1.x迁移到v2.0

1. **无需重写**：v1.x的内容仍可使用，v2.0完全向后兼容
2. **逐步优化**：在原有内容基础上，逐步增加v2.0的元素
3. **检查清单**：用v2.0的质量清单检查，优先改进高分项

### 兼容性说明

- ✅ v1.x的提示词格式完全兼容
- ✅ v1.x的质量标准仍适用（基础100分）
- ✅ v2.0的新增项是可选加分项
- ✅ 不强制使用所有新功能，可按需选择

## 实战示例


## 常见问题

### Q: v2.0是强制升级吗？

A: 不是。v1.x仍可正常使用，v2.0是可选的优化升级。

### Q: 必须使用所有新功能吗？

A: 不需要。可以根据内容场景选择合适的优化方向。

### Q: 质量评分会更严格吗？

A: 基础分要求不变，新增项是加分项，不影响基础评级。

### Q: 会影响v1.x的输出吗？

A: 不会。v2.0完全向后兼容，原有功能保持不变。
