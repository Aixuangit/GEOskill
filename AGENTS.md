## 输出底线

### 一票否决项（绝不能碰）

❌ 有虚假数据/案例
❌ 有违规敏感词
❌ 有极限词且无法替换
❌ 有联系方式（微信、电话、二维码）
❌ 内容完全重复，无信息增量
❌ 直接违反广告法
❌ 有政治/色情/暴力内容
❌ 侵犯版权/知识产权

---

### 必须包含项（缺一不可）

✅ 场景判断报告
✅ 关键词规划
✅ 5个标题备选
✅ 完整文章大纲
✅ 正文（符合场景要求）
✅ 配图方案（3-5张）
✅ 质量自检结果

---

### 质量底线（必须达到）

✅ 质量审查总分 ≥ 75分（良好及以上）
✅ GEO适配度 ≥ 20分
✅ E-E-A-T合规 ≥ 20分
✅ 合规检查必须满分（20分）
✅ 去AI味检查 ≥ 15分

## 新增场景要求

### 新增场景的完整步骤

1. **场景定义**
   - 场景名称
   - 适用需求说明
   - 推荐字数
   - 默认插图风格

2. **场景模板**
   - 角色设定
   - 写作规则
   - 标题公式（至少3个）
   - 开头示例
   - 正文结构
   - 结尾示例

3. **更新相关文件**
   - `skill/SKILL.md`：添加新场景到场景列表
   - `skill/docs/illustration_style_system.md`：确认插图风格匹配
   - `skill/templates/scene_output_template.md`：添加新场景输出示例
   - `skill/examples/usage_examples.md`：添加新场景调用示例
   - `skill/examples/example_output_xxx.md`：添加完整输出示例（可选）

4. **测试验证**
   - 测试新场景是否正常工作
   - 检查质量是否达标
   - 验证合规性

5. **文档更新**
   - 更新 CHANGELOG
   - 更新 README
   - 更新本文档（如有需要）

## 发布流程

### 版本号规则

使用语义化版本号：`v[主版本].[次版本].[修订号]`

- **主版本**：重大功能变更，不向后兼容
- **次版本**：新增功能，向后兼容
- **修订号**：Bug修复、文档更新

### 发布步骤

1. **更新文档**
   - 更新 CHANGELOG
   - 更新 README
   - 更新版本号

2. **测试验证**
   - 完整测试所有功能
   - 确认质量达标

3. **发布**
   - Git commit
   - Git tag（版本号）
   - Git push

## FAQ

### Q: 如何提交问题或建议？

A: 可以通过以下方式：
- GitHub Issues（如有仓库）
- 邮件反馈
- 群内讨论

### Q: 多久更新一次？

A: 计划：
- 重大更新：每季度1次
- 小更新：每月1次
- 紧急修复：随时

### Q: 可以自定义场景吗？

A: 可以。按照"新增场景要求"的步骤添加自定义场景，建议贡献回社区。

### Q: 如何适配其他平台？

A: 可以在 `skill/agents/` 目录下添加新的平台配置文件，如 `github.yaml`、`notion.yaml` 等。
